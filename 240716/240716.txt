디벨롭은 조원들의 코딩 병합.
마스터는 최종. = 피엠만 만지기.
픽쳐-a는 조원

픽쳐a(조원들이) 
피엠이 폴더만들었으면 조원들은 자기 컴퓨터에 새폴더만들고 코드열기.
내려 받아야하는데 방법이 두가지 있다.
1. git pull origin master (나의 로컬컴퓨터에 저장된 데이터도 살리고 원격 저장소에 커밋된 데이터도 가져와서 병합을 하겠다.)

2.git clone "레포지스토리주소" (원격 저장소에 커밋된 데이터를 전부 가져와서 시작한다.) = 내컴퓨터에 있는건 다 덮어쓰니까 맨처음에 내려받을때만 쓴다.
홈페이지가서 코드에 hhtps 복사.
터미널에서 진행.
클론하고나서 다시 나갓다 오기.
그리고 .git이 있는 폴더에 .git폴더로 들어가서 코드열기.

피엠또는 다른 조원이 가져온 파일 작업하고.
커밋하려면 git checkout feature-a(조원파일이름)
git add .

팀원이 해야하는 것.
피엠이 초대해준 메인창에 상단보면 pull requests 클릭.
new pull requset 클릭(초록색 버튼)
반드시 베이스는 develop / commoare는 featur-a(조원작업 파일 명)
하면 추가로 깃허브가 새로 생성된 코드를 색으로 표현해줌.
호가이노디면 초록버튼 크레에이티브 풀 리퀘스트 클릭.
이후 제목에 marge요청, 내용 입력~~` 병합요청드립니다 등등 피엠에게 안내말 건네기.
피엠님께서는 확인하게되면 pull requests에서 파일 채인지드 클릭하면 어디가 변경된지 확인할수잇다.
피엠님게서 병합승인해주면 팀원이 다시 들어간다.
승인된것을 확인하면 
마진 풀 리퀘스트 클릭하면 피엠이 다시 체킹.
